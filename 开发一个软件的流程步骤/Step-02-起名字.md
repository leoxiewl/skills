# Step 02 — 起名字

> [!abstract]
> 名字确定了，项目才"存在"。一个好名字是项目文件夹、git repo、域名、App Store 页面的共同基础。

## 命名原则

**风格选择**（选其一即可，不要混）：

| 风格 | 例子 | 适合场景 |
|---|---|---|
| 极简英文单词 | Perch / Helm / Peak | 有上架 App Store 打算，想要品牌感 |
| 功能直白 | FocusBar / TopBar / NudgeBar | 个人工具，让人一眼知道是干什么的 |
| 中文或中英混合 | 总控 / 聚焦栏 | 纯个人使用，无上架打算 |

> [!tip]
> 功能直白的名字适合独立开发早期——用户（包括自己）能立刻理解它做什么，减少解释成本。

## 命名流程

1. 先确定风格偏好
2. 列出 3-5 个候选，逐一感受"念出来顺不顺"
3. 快速决策，不要纠结——名字可以改，但越早定越好
4. 确定项目文件夹命名规范（推荐全小写，无空格）

## FocusBar 的命名过程

**候选清单（功能直白风格）**：
- FocusBar — 专注栏，常驻 menu bar，帮你专注
- TopBar — Top = 首要事情 + 屏幕顶部，双重含义
- NudgeBar — Nudge = 轻轻提醒你该干什么了
- PriorBar — Prior = Priority 缩写，优先级栏

**最终选择**：`FocusBar`

**理由**：
- "Focus" 直接命中核心功能（专注、聚焦）
- "Bar" 点明形态（menu bar app）
- 简洁，好记，读起来流畅

**项目文件夹名**：`focusbar`（全小写，方便命令行和 git）

> [!note]
> 项目名、git repo 名、文件夹名统一用 `focusbar`。App 显示名称用 `FocusBar`（首字母大写）。
