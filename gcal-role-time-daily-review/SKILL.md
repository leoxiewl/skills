# 日历总结 Skill

从 Google Calendar 读取指定日期的所有事件，按角色日历分类汇总时间和占比，生成结构化表格，追加到当天日记文件末尾。工具：`gws` Calendar 读取能力 + `calendar_summary.py` 脚本。

> **边界**：只读取日历数据，不写入任何事件；只统计有明确起止时间的事件；未记入日历的时间不会出现。

---

## Step 1 — 确定目标日期 & 检查认证

用户可以说"今天""昨天""2026-06-24"——用系统日期换算为绝对日期，不靠猜。

```bash
gws auth status
```

- `auth_method: none` → 提示用户运行 `gws auth login`，**终止**
- 认证正常 → 继续

---

## Step 2 — 运行脚本

脚本路径：`_my/my-skills/gcal-role-time-daily-review/scripts/calendar_summary.py`（相对于 vault 根目录）

```bash
# 预览：只输出 Markdown 到终端
python3 _my/my-skills/gcal-role-time-daily-review/scripts/calendar_summary.py --date YYYY-MM-DD

# 写入日记文件
python3 _my/my-skills/gcal-role-time-daily-review/scripts/calendar_summary.py --date YYYY-MM-DD --write
```

脚本自动完成：
- 调用 `gws` 获取日历列表和事件数据
- 解析起止时间；全天事件标注"全天"计 0h；跨日事件截断到 00:00–24:00
- 生成全部事件列表（按开始时间排序，含时间段、时长、事件名、所属日历）
- 按角色合并（见下方规则）
- 计算占 24h 比例，生成信号灯
- `--write` 模式下自动定位日记文件并追加（路径规则见下方）

**如果脚本不可用**，回退到手动模式（Step 2a）。

---

## Step 2a — 手动模式（脚本不可用时）

### 获取日历列表

```bash
gws calendar calendarList list --page-all
```

从返回 JSON 的 `items` 中提取每个日历的 `summary`（名称）和 `id`（calendarId）。

### 逐日历查询事件

```bash
gws calendar events list \
  --params '{"calendarId": "CALENDAR_ID", "timeMin": "YYYY-MM-DDT00:00:00+08:00", "timeMax": "YYYY-MM-DDT23:59:59+08:00", "singleEvents": true, "orderBy": "startTime"}'
```

对每个事件提取 `summary`、`start.dateTime`、`end.dateTime`，计算时长。

## 角色合并规则

| 角色类别 | 合并来源 | 信号灯阈值 |
|---|---|---|
| 🌿 健康 | 🏃🏻‍♂️运动 + 💤睡眠 | ✅ ≥7h / 🟡 ≥5h / ⚠️ <5h |
| 🧭 Founder | 🧭 Founder | ✅ ≥1h / 🟡 >0h / 🔴 0h |
| 💼 职业工作 | 职业工作 | ✅ ≥4h / 🟡 ≥2h / ⚠️ <2h |
| 🌍 生活 | 生活 + 娱乐 | — |
| 🤝 关系 | 伴侣 + 朋友 + 下一代 + origin 家庭 | 🔴 任一角色 0h |
| 🧠 系统 | CEO(Sys) | — |

跳过：月相、中国节假日（不参与统计）。

## 日记写入规则

路径各部分必须从**目标日期**动态计算，禁止硬编码：

| 占位符 | 含义 | 计算方式 |
|---|---|---|
| `YYYY` | 年份 | 目标日期的年，如 `2026` |
| `YYYY-MM` | 年月 | 目标日期的年月，如 `2026-06` |
| `YYYY-WNN` | ISO 周 | 目标日期的 ISO 周数，两位补零，如 `2026-W26` |
| `YYYY-MM-DD` | 日期 | 目标日期，如 `2026-06-25` |
| `星期X` | 中文星期 | 目标日期对应的中文星期，如 `星期四` |

完整路径模板：
```
个人OS/02_Reviews/{YYYY}/{YYYY-MM}/{YYYY-WNN}/{YYYY-MM-DD}-{星期X}.md
```

示例（目标日期 = 2026-06-25）：
```
个人OS/02_Reviews/2026/2026-06/2026-W26/2026-06-25-星期四.md
```

- 追加到文件末尾（`## 相关` 区域之前）
- 如果已有 `## 📊 日历时间数据` 区域，**替换**旧数据（同一日期只保留最新）
- 如果日记文件不存在，只输出到对话，不自动创建

## 异常处理

| 情况 | 处理 |
|---|---|
| 认证失败 | 提示 `gws auth login`，终止 |
| 某日历查询失败（403/404） | 跳过，标注"[查询失败]"，继续其他日历 |
| 当天无任何事件 | 输出空表格，标注"当天无日历事件记录" |
| 全天事件 | 标注"全天"，时长计 0h |
| 跨日事件 | 截断到目标日期 00:00–24:00 |
| 事件时间重叠 | 各日历原样计算，标注"[时间重叠]" |
| 日记文件不存在 | 只输出到对话，不创建文件 |
